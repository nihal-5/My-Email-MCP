### Reporting Bugs

Before creating bug reports, please check the existing issues. When you create a bug report, include as many details as possible:

- **Use a clear and descriptive title**
- **Describe the exact steps to reproduce the problem**
- **Provide specific examples**
- **Describe the behavior you observed and expected**
- **Include screenshots if applicable**
- **Note your environment** (OS, Node.js version, etc.)

### Suggesting Enhancements

Enhancement suggestions are tracked as GitHub issues. When creating an enhancement suggestion:

- **Use a clear and descriptive title**
- **Provide a detailed description of the suggested enhancement**
- **Explain why this enhancement would be useful**
- **List any similar features in other projects**

### Pull Requests

1. Fork the repo and create your branch from `main`
2. If you've added code that should be tested, add tests
3. Ensure the test suite passes
4. Make sure your code lints
5. Issue that pull request!

## ğŸ“ Development Process

1. **Clone the repository**
   ```bash
   git clone https://github.com/nihal-5/My-Email-MCP.git
   cd My-Email-MCP
   ```

2. **Install dependencies**
   ```bash
   npm install
   ```

3. **Create a branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

4. **Make your changes**
   - Write clean, readable code
   - Follow TypeScript best practices
   - Add comments for complex logic

5. **Test your changes**
   ```bash
   npm run build
   npm test
   ```

6. **Commit your changes**
   ```bash
   git commit -m "feat: Add amazing feature"
   ```

7. **Push and create PR**
   ```bash
   git push origin feature/your-feature-name
   ```
